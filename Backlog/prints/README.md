# Prints — Screenshots de consulta

Pasta para guardar screenshots e imagens que sirvam de contexto para itens de backlog: telas de plataforma, gráficos, exemplos de configuração, capturas de conversa.

## Convenção de nome

`YYYY-MM-DD_<sistema>_<assunto>.<ext>`

Exemplos:
- `2026-05-22_bitrix_funil-cobranca.png`
- `2026-05-22_torre_dashboard-executive.png`
- `2026-05-22_esperanza_conversa-promessa.png`

## Como referenciar

Item de backlog referencia prints no corpo (não no frontmatter):

```markdown
## Contexto visual

Ver `Backlog/prints/2026-05-22_bitrix_funil-cobranca.png` — fluxo atual antes da nova raia de reembolso.
```

## Recomendação

Use prints para o que **texto não captura bem**: layout, hierarquia visual, comportamento dinâmico capturado em frame. Não use para conteúdo que poderia ser texto.
