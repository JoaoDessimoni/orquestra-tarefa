# Contexto — Docs de apoio para o backlog

Pasta com documentação técnica e de negócio que serve de **referência viva** para os itens de backlog. Não são solicitações (`solicitacoes/` para isso), nem são fontes de verdade do negócio inteiro (`Docs/finza/` para isso) — são docs específicos que orientam refinamento e priorização de backlog.

## Arquivos atuais

| Arquivo | Frente principal | Origem |
|---|---|---|
| `torre_de_controle_overview.md` | Torre | Migrado de `Docs/finza/TORRE_DE_CONTROLE_OVERVIEW.md` |
| `esperanza_agent_overview.md` | Esperanza | Migrado de `Docs/agentes/ESPERANZA_AGENT_OVERVIEW.md` |

## Quando adicionar arquivo aqui

- Doc técnico de uma plataforma que apoia múltiplos itens de uma frente (ex: spec da nova carteira X)
- Resumo de discovery feito pelo squad (output de pendência discovery)
- Glossário/dicionário operacional específico do backlog
- Diagramas de fluxo que apoiem refinamento

## Quando NÃO adicionar aqui

- Contexto de negócio amplo da Finza → `Docs/finza/`
- Solicitação formalizada pelo gestor → `Backlog/solicitacoes/`
- Print/screenshot → `Backlog/prints/`
- Nota de reunião → `Gestao/Reunioes/`
